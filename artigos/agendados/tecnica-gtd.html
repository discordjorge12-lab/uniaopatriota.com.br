GTD (Getting Things Done): o método de David Allen explicado de forma simples

O GTD é um dos sistemas de produtividade mais completos já criados — e um dos mais mal compreendidos. Aqui está o que realmente importa no método.

Foto: Picsum Photos via Unsplash

O problema que o GTD resolve

David Allen criou o GTD para resolver um problema específico: a mente humana é péssima para guardar coisas, mas ótima para processá-las.

Quando você tem itens pendentes guardados na cabeça, eles criam o que Allen chama de 'loops abertos' — pensamentos recorrentes que consomem energia mental sem avançar nada. O GTD resolve isso colocando tudo para fora da cabeça em um sistema confiável.

Os 5 pilares do GTD

1. Capturar: Tudo que exige sua atenção vai para uma 'caixa de entrada' — pode ser um app, um caderno, um e-mail para si mesmo. Qualquer lugar, desde que você confie que vai processar depois.

2. Processar: Regularmente, você esvazia a caixa de entrada. Para cada item: pode ser feito em 2 minutos? Faz agora. Se não, delega, agenda ou coloca na lista de projetos.

3. Organizar: O que não foi feito vai para o lugar certo: lista de próximas ações, lista de projetos, algum dia/talvez, ou referência.

4. Revisar: Semanalmente, você revisa todo o sistema — garante que está atualizado e que nada ficou para trás.

5. Engajar: Com o sistema em ordem, você trabalha sabendo que está fazendo a coisa certa no momento certo.

O que é uma 'próxima ação'?

Um projeto como 'lançar novo produto' não tem como ser feito — é grande demais.

O GTD exige que todo projeto tenha pelo menos uma 'próxima ação física' definida: a menor ação possível para avançar o projeto. Ex: 'Enviar e-mail para João perguntando sobre fornecedor X'.

Quando você sabe qual é a próxima ação, a procrastinação diminui. A tarefa nebulosa vira algo concreto e fazível.

A revisão semanal — a peça central

A revisão semanal é onde o sistema se mantém vivo. Sem ela, o GTD vira mais um monte de listas esquecidas.

Em 30 a 60 minutos por semana: esvaze caixa de entrada, revise projetos ativos, revise lista de próximas ações, olhe agenda da semana seguinte. No final, você tem clareza total do que está em aberto.

Versão simplificada para começar

Você não precisa implementar o GTD completo. Comece com: uma caixa de entrada (captura tudo), uma lista de próximas ações por contexto (ex: @computador, @telefonemas, @compras) e uma revisão semanal.

Isso já representa 80% dos benefícios do sistema completo.

Conclusão

O GTD não é uma religião — é um framework que você adapta. O princípio central vale para qualquer sistema: tire tudo da cabeça, processe regularmente, mantenha apenas próximas ações concretas nas listas. A mente livre de loops abertos pensa melhor.
